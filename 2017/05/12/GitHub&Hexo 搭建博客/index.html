GitHub&Hexo 搭建博客

发表于 2017-05-12 | 分类于 Technology

初入公司，第一个任务就是自己搭建一个博客。其实之前就一直想试一试自己搭建个博客，但出于懒，一直没有动手。直到，刚入职场，公司老板布置了搭建技术博客的任务，才正式对待这个问题。废话不多说，下面就用Github➕ Hexo的方法搭建一个简单的、属于自己的、可定制的博客。

说明：

本文主要针对Mac版本，windows版在文章的结尾有简要的说明，但各种差异请自行甄别。

环境

本文利用Github Pages进行远端部署博客后台，用Node.js的一个命令行工具hexo生成博客所需的静态页面。

1. 安装Gitb

搜索git for mac／windows,根据你的操作系统安装不同版本的git，建议地址:Git for Mac

2. 安装Node.js

可以去Node.js的官网Node.js下载最新版本,记住：必须是最新版。Mac下面可以先安装Home brew包管理器，然后用命令brew install node安装最新版
如果能够正常使用命令npm，代表安装成功。

3. 安装Hexo

安装完Node.js以后，就可以用Node自带的包管理安装hexo如下：

npm install hexo-cli -g

如果能够正常使用hexo命令，代表安装成功，直接敲hexo就可以了。

至此，环境已经搭建完成，接下来是对环境的补充说明，觉得没有必要的可以掠过这段。但很多人在搭建自己的博客的时候对各个需要搭建的环境并不清晰，我这里简单地理一下关系。

1. Git主要是将本地生成的静态页面部署到远端
2. Node.js是Javascript的运行平台，这里主要是hexo生成静态页面的引擎
3. Hexo是Node.js的一个命令行工具，将Markdown编译为静态页面的工具

Hexo命令和配置

Hexo 几个常见的命令：

  1. hexo init [blogname]生成博客，表现为本地的一个文件夹，文件夹名字为[blogname]
  2. hexo new [passagename]生成一篇博文，可以在[blogname]/source/_posts下面找到名为[passagename].md的文件，
  3. hexo generate[g]生成静态页面，一般在hexo new命令以后
  4. hexo deploy[d]将本地代码部署到远端（这里指Github），需要在配置以后才能进行

Hexo 配置说明：
在文件夹[blogname]下面，可以看到有个_config.xml的文件，关于博客的主要配置都在这个文件里面。这里重点关注以下几个方面：

1. 博客的主要信息：标题、副标题、描述、作者、语言和时区
title: Geek
subtitle:
description:
author: Huanjian Jiang
language:
timezone:



2. git配置信息：远程类型、仓库地址、分支和提交信息
deploy:
type: git
repo: git@github.com:zhuwo/huanjian.git
branch: master
message: Final try

3. 模版
你可以通过搜索引擎找到网上大量的网页模版，然后将模版的整个文件夹放到[blogname]/themes文件夹下面，然后将theme 后面的参数设置为你想要的模版对应的文件夹名。
theme: master

过程：

  1. hexo init 生成博客
  2. 修改配置文件_config.xml
  3. hexo deploy 部署到Github

注意点：

1. 每次添加markdown文件以后都要重新生成静态页面 
2. 每次本地操作以后都要重新部署到github
3. 配置项目的：后必须跟一个空格，比方说theme:[空格][模版名]

Github

作为一个程序员，github的使用是必须的，这里并不累赘复述如何注册github账号，如何在github创建仓库等等。只说明这里创建博客的注意点。

1. 新建一个仓库后，仓库名必须与GitHub帐户一致，比方说的我的github地址为https://github.com/zhuwo,那么仓库名必须为zhuwo.github.io。这样，最后你的博客地址也就是https://zhuwo.github.io。
2. 本地部署到github以后，再在仓库的设置页面，将分支改为master（仓库没有数据的时候不能选择分支，也就没有github page）

最后，可以在仓库的设置页面看到github page的地址，就是你的博客地址。欢迎大家访问我的博客Geek
